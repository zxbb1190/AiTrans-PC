# AiTrans 0.2.3

## 中文说明
- 引入 `paddleocr_local -> tesseract -> windows_ocr_api` 本地 OCR 链，明显提升中英日截图文本的识别准确率。
- 为本地 PaddleOCR 加入常驻 worker 与启动预热，降低长块文本和后续连续截图的等待时间。
- 调整 OCR 自动语言判断、英文单行与宽块英文优先策略，减少中文模型抢跑带来的误识别。
- 设置与连接现在支持 OpenAI、DeepSeek、智谱 AI 与其他 OpenAI-compatible 服务，并支持保存模型名；同时修复设置页重复保存时 `API Key` 被清空的问题。
- 翻译失败时会在对话中直接展示用户可理解的错误原因，不再把 stub 回退结果误当作正常成功。
- 对话体验继续打磨：新聊天与聊天历史管理分离、滚动跟随逻辑更合理、截图预览支持直接复制图片。

## English Notes
- Introduces a local OCR chain of `paddleocr_local -> tesseract -> windows_ocr_api`, significantly improving screenshot OCR accuracy across Chinese, English, and Japanese text.
- Adds a persistent local PaddleOCR worker plus startup prewarm to reduce waiting time for long text recognition and repeated captures.
- Refines OCR auto-language routing and English-first heuristics for single-line and wide English blocks, reducing misrecognition caused by non-English-first attempts.
- The setup flow now supports OpenAI, DeepSeek, Zhipu AI, and other OpenAI-compatible services, including configurable model names, and fixes the issue where saving settings could accidentally clear the `API Key`.
- Translation failures are now surfaced directly in the chat UI with user-readable reasons instead of being silently presented as stub success.
- Continues polishing the chat experience: new chat is separated from history management, auto-follow scrolling is more predictable, and screenshot previews can now be copied as images.
