# AiTrans 0.1.2

## 中文说明

### 变更摘要

- 发布 `0.1.2` 版本，用于验证 Windows 安装版自动更新链路
- 对齐 `stable` 更新通道的元数据产物与发布校验
- 完整化 Windows 发布脚本，支持自动物化项目、清理旧产物并生成发布 manifest
- 修正版本对齐校验，避免 `product_spec`、Electron 包版本与 release notes 脱节

### 对使用者可见的影响

- Windows 安装版现在可以更稳定地参与自动更新测试
- `release:win` 生成的发布目录会自动清理旧版本残留
- 更新元数据文件按 `stable` 通道生成 `stable.yml`
- `portable` 便携版仍通过重新下载新包完成升级

### 安装或获取方式

- 安装版：运行 `desktop-screenshot-translate-0.1.2-x64.exe`
- 便携版：运行 `desktop-screenshot-translate-0.1.2-x64-portable.exe`
- 若要测试自动更新，请为已安装版配置：
  - `release.update_base_url`
  - 或 `AITRANS_UPDATE_BASE_URL`

### 已知限制

- 当前公开交付仍以 Windows 为主，macOS 尚未进入正式发布范围
- 当前未完成代码签名，部分 Windows 环境可能出现额外安全提示
- 若未提供可达的更新源，自动更新会保持禁用并回退为手动升级
- 若未提供可达的翻译端点与凭据，应用会退回本地 stub 翻译

## English Notes

### Summary

- Release `0.1.2` to validate the Windows installed-build auto update flow
- Align release validation with the `stable` channel metadata output
- Complete the Windows release scripts so they materialize the project, clear old artifacts, and generate a release manifest
- Fix version alignment checks across `product_spec`, the Electron package, and release notes

### User-visible impact

- The Windows installed build is now better prepared for auto update testing
- `release:win` now clears stale files from the release directory before building
- Update metadata is now expected as `stable.yml` for the `stable` channel
- The portable build still upgrades by downloading a new package manually

### Install / Download

- Installer: run `desktop-screenshot-translate-0.1.2-x64.exe`
- Portable: run `desktop-screenshot-translate-0.1.2-x64-portable.exe`
- To test auto update, configure:
  - `release.update_base_url`
  - or `AITRANS_UPDATE_BASE_URL`

### Known limitations

- Public delivery still targets Windows first; macOS is not part of the formal release scope yet
- Code signing is not finished yet, so some Windows environments may show additional security prompts
- If no reachable update feed is configured, auto update stays disabled and the app falls back to manual upgrade
- If no reachable translation endpoint and credential are provided, the app falls back to local stub translation
